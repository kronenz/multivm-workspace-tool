# Design Principles — 디자인 원칙

> **역할**: UI/UX Designer, Frontend Engineer
> **적용 범위**: UI 컴포넌트, 시각 디자인, 인터랙션 패턴

---

## 핵심 원칙

### 1. 터미널 중심 UI

이 앱의 주인공은 **터미널**이다. 모든 UI 결정은 터미널 가독성과 사용성을 최우선으로 한다:

- 터미널 영역이 전체 화면의 70% 이상을 차지해야 한다
- 사이드바, 상태바 등 보조 UI는 최소화하되 접근 가능해야 한다
- xterm.js WebGL 렌더러를 기본으로 사용, Canvas 폴백 제공

### 2. Dark-First, Light-Ready

- **다크 모드**를 기본 테마로 설계한다 (개발자 80%+ 선호)
- **라이트 모드**는 동등한 품질로 제공한다 (MVP Feature 10)
- 테마 전환은 즉시 적용 (앱 재시작 불필요)
- 모든 UI 컴포넌트에 일관되게 적용

### 3. 그리드 레이아웃 직관성

| 프리셋 | 용도 | 대상 |
|--------|------|------|
| 1x1 | 단일 VM 집중 | Solo Dev, 단일 작업 |
| 2x1 | 좌우 비교 | Dev + Staging 비교 |
| 2x2 | 4개 VM 동시 | Microservices 병렬 작업 |
| Custom NxM | 자유 분할 | 5개+ VM 고급 사용자 |

- 패인 리사이즈는 드래그로, **50ms 이내** 응답 (NFR-3)
- 활성 패인은 시각적으로 구분 (컬러 보더)
- 레이아웃 상태는 Workset에 자동 저장/복원

### 4. 정보 밀도 최적화

리소스 모니터링 표시:

```
VM 1: CPU 45% ● RAM 62% ● Disk 78% ●
       (green)   (yellow)   (yellow)
```

| 범위 | 색상 | 의미 |
|------|------|------|
| 0-49% | 🟢 초록 | 정상 |
| 50-79% | 🟡 노랑 | 주의 |
| 80-100% | 🔴 빨강 | 위험 |

### 5. 접근성 기본 준수

- 색상만으로 정보를 전달하지 않는다 (숫자 값 병기)
- 키보드 네비게이션으로 모든 핵심 기능 접근 가능
- 충분한 대비비 (WCAG AA 수준)
- 폰트 크기 조절 가능

### 6. 일관된 인터랙션 패턴

| 패턴 | 적용 |
|------|------|
| 클릭 | 선택, 활성화, 토글 |
| 더블클릭 | 열기, 확장 |
| 드래그 | 리사이즈, 이동 |
| 우클릭 | 컨텍스트 메뉴 |
| Esc | 취소, 닫기 |

## 컴포넌트별 디자인 가이드

| 컴포넌트 | 핵심 고려사항 |
|----------|---------------|
| Terminal | xterm.js 기본 테마 준수, 256색/truecolor, 10K 스크롤백 |
| File Browser | 트리 뷰, 아이콘 구분 (폴더/파일), 읽기 전용 명시 |
| Markdown Viewer | 코드 블록 구문 강조, 5초 자동 새로고침 |
| Workset Sidebar | 검색/필터, CRUD 버튼, 드래그 정렬 |
| Resource Monitor | 컬러 코딩, 숫자 값, 5초 갱신 |
